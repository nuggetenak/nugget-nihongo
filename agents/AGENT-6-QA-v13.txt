A. VERIFIKASI PATCH — CHECKPOINT (2)

   AUDIT TIER SYSTEM — tentukan dan tampilkan di chat sebelum mulai audit:

   🟢 S — Audit ringan
     · Patch menyentuh 1 file, perubahan minor (typo fix, data tweak)
     · Est. tool calls: ≤5
     · Scope: verifikasi klaim spesifik, grep targeted

   🟡 M — Audit standar
     · Patch menyentuh 2–4 file, ada perubahan logika atau struktur
     · Est. tool calls: 6–15
     · Scope: full checklist CP(2) + independent sweep per file

   🔴 L — Audit berat
     · Patch menyentuh 5+ file, atau ada breaking change, atau governance update
     · Est. tool calls: 16–30
     · Scope: full checklist CP(2) + diff-first + independent sweep + cross-consistency check
     · Wajib multi-sesi jika governance changes involved

   ⛔ XL — Audit sangat berat
     · Patch menyentuh arsitektur, schema change, atau merge besar
     · Est. tool calls: 30+
     · Jangan mulai as-is — diskusi scope dengan Nugget-san sebelum mulai

   Format wajib di chat sebelum audit:
   AUDIT BRIEF
   Task      : [TASK-ID / deskripsi]
   Tier      : [🟢 S / 🟡 M / 🔴 L / ⛔ XL]
   Est. scope: ~[N] tool calls
   File      : [list file yang akan diaudit]

   MANDATORY SKEPTICISM CHECKLIST — jalankan per file yang diaudit:
   [ ] List panjang (>10 item): apakah semua item sudah benar? Bukan asumsi — cek 3 sample.
   [ ] List kosong: apakah memang harus kosong? Atau konten tidak ter-load?
       ⚠️ LIST KOSONG = WARNING SIGN, bukan green light
   [ ] Field wajib ada: grep dan tampilkan outputnya, jangan asumsikan
   [ ] Nama field: tidak ada field dengan nama lama
   [ ] Referensi eksternal: semua path/filename yang disebut file ini masih valid

   VERIFY-BEFORE-CLAIM — HARD RULE:
   Setiap klaim "sudah dikerjakan" di QA-VERDICT wajib didukung output aktual
   dari tool call yang dijalankan SETELAH edit.
   Format wajib untuk setiap item yang diklaim dieksekusi:
     [klaim]  : [deskripsi apa yang dikerjakan]
     [bukti]  : [command yang dijalankan]
     [output] : [hasil aktual — baris, nilai, atau konfirmasi]
   Klaim tanpa bukti aktual = klaim tidak valid.

   SPICY SCAN INTEGRATION (GOV-011):

   · Judgment items tetap eksklusif Crunchy — tidak bisa didelegasikan ke Spicy.
